## Goal

Replace every occurrence of `settings.default_dealership_id` in admin routes with `did = await auth_check(request)`. After 06-01 lands, `auth_check` returns either a `RedirectResponse` (unauthenticated) or an `int` (dealership_id). This plan updates all admin route handlers to use the `int` returned from that check instead of the hardcoded global default.

After this plan: each admin session is fully scoped to one dealership; a user logged in as dealership 2 cannot see dealership 1's data.

**Deployment constraint:** This plan must be applied in the same session as 06-01. Do not start the server between 06-01 and 06-03.

---

## Dependencies

- **06-01** must be complete: `auth_check` in `admin_common.py` must return `int | RedirectResponse`.

---

## Pre-work: understand the call pattern change

After 06-01, the correct call pattern in every route function is:

```python
did = await auth_check(request)
if not isinstance(did, int):
    return did
```

Use `isinstance(did, int)` uniformly across all routes. Remove any `redir = await auth_check(request); if redir: return redir` patterns — replace them entirely with the `isinstance` guard.
