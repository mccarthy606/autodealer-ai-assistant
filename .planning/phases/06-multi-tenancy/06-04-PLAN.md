### Task 1 — Extend `conftest.py` fixtures for multi-tenant scenarios

**File:** `tests/conftest.py`

**BLOCKER 7 FIX:**

Before modifying `conftest.py`, take these steps:

**Step A — Audit existing tests that use the `dealership` fixture:**
Run `grep -rn "def test_.*dealership\|fixture.*dealership\|dealership.*fixture" tests/` and `grep -rn "def dealership\|async def dealership" tests/conftest.py` to understand what fields the current `dealership` fixture provides. Then run `grep -rn "\bdealership\b" tests/` to list every test file that uses it. For each test file found, read it and note whether it accesses `dealership.admin_username`, `dealership.admin_password_hash`, or `dealership.whatsapp_access_token`. These are the new fields — existing tests that do not access them will be unaffected by the fixture update.

**Step B — bcrypt performance:** Do NOT call `bcrypt.hashpw(b"pass1", bcrypt.gensalt())` inside the fixture body. `gensalt()` is CPU-intensive. If called per-test, it will slow the suite significantly. Instead, compute the hash once at module level as a constant:

```python
import bcrypt

# Computed once at module load — bcrypt is slow per-call
_DEALER1_HASH: str = bcrypt.hashpw(b"pass1", bcrypt.gensalt()).decode()
_DEALER2_HASH: str = bcrypt.hashpw(b"pass2", bcrypt.gensalt()).decode()
```

Place these constants at the top of `conftest.py` (after imports). Reference them from the fixtures.

**Step C — Update existing `dealership` fixture (do not replace — update):**

The existing fixture already creates a `Dealership` row. Add the three new multi-tenant fields to it. Keep all existing fields exactly as they are. If the existing fixture does not have `whatsapp_phone_number_id` or `ml_user_id`, add them now (required by 06-04 tests):
